What is normalization, standardization and scaling?



Normalization, standardization, and scaling are all techniques used in data preprocessing to transform your data into a common scale. This is important for many machine learning algorithms because they can be sensitive to the ranges of the features (columns) in your data. Here's a breakdown of each technique:
Normalization:
    

  • Goal: Normalize data points to a specific range, typically between 0 and 1 (or -1 and 1).
    • 

  • Method: It applies a linear transformation to each feature independently. This means it subtracts the minimum value of the feature from each data point and then divides by the difference between the maximum and minimum values.
    • 

  • Use Case: Normalization is useful when you want all features to contribute equally to the model, regardless of their original units. It's also helpful when the data contains outliers that might significantly affect other techniques.
    • 

Standardization:
    

  • Goal: Transform data to have a standard normal distribution (also known as a Gaussian distribution) with a mean of 0 and a standard deviation of 1.
    • 

  • Method: Similar to normalization, it subtracts the mean of the feature from each data point and then divides by the standard deviation. This centers the data around 0 and scales it based on the spread of the data.
    • 

  • Use Case: Standardization is often preferred when the features in your data have different scales and the distribution of the data is important for the machine learning algorithm.  For example, some algorithms like Support Vector Machines (SVMs) rely on distances between data points, and standardization ensures these distances are comparable across features.
    • 

Scaling:
    

  • General Term: Scaling is a broader term that encompasses both normalization and standardization. It refers to any technique that transforms features to a common scale.
    • 

Here's a table summarizing the key differences:


FeatureNormalizationStandardization
GoalSpecific range (often 0-1 or -1-1)Standard normal distribution (mean 0, std 1)
MethodLinear transformationSubtract mean, divide by standard deviation
Use CaseFeatures with different units, outliersFeatures with different scales, distribution matters


Choosing the Right Technique:
The choice between normalization and standardization depends on your specific data and the machine learning algorithm you're using. Here are some general guidelines:
    

  • Use normalization if you want all features to have equal weight and the presence of outliers is a concern.
    • 

  • Use standardization if the distribution of your data is important for the algorithm and features have different scales.
    • 

  • If you're unsure, standardization is often a good default choice.
    • 

Remember, the goal is to transform your data into a format that allows your machine learning algorithm to learn effectively. Experiment with different scaling techniques and see what works best for your specific problem.

Here's an example code snippet using Python for IQR scaling on 1D time series data:

import numpy as np

def iqr_scaling(data):
    """
    Scales time series data using Interquartile Range (IQR) scaling.

    Args:
        data (np.array): 1D time series data.

    Returns:
        np.array: Scaled time series data.
    """
    q1 = np.percentile(data, 25)  # Lower quartile
    q3 = np.percentile(data, 75)  # Upper quartile
    iqr = q3 - q1  # Interquartile range

    # Handle potential zero IQR (all values the same)
    if iqr == 0:
    return data  # No scaling needed

    scaled_data = (data - q1) / iqr
    return scaled_data
